传播博客

Kaitou KID

发布于：2020年6月24日

我的博客是用Github Page和Hexo搭建的。至少在做这篇博客时，我并不是什么计算机大神，对软工方面的知识也只限于略懂一些。所以，搭建博客并不是特别复杂的事情，只要认真查阅资料，就可以建立起一个免费而有趣的博客。

特此感谢我的好友鸡王先生提供的建议，本来我是准备用亚马逊云服务器做博客的，但是管理服务器很复杂，听从鸡王先生的建议，转战Github Page。备注：主机是Windows 10系统。Linux系统和IOS系统应该比Windows系统简单一些，大同小异。

预备元素

  • 常用Bash命令知识
  • Github账户和基础知识

主机准备

  • 安装NodeJS

    • 下载入口

  • 安装Git

    • 下载入口
    • 参考：《GitHub入门与实践》
    • Git 安装成功后，右键菜单栏会有 Git Bash here栏

  • 安装Hexo

    • 选择一个上级目录（可以是桌面）打开Git Bash

    • npm install -g hexo-cli
hexo init “自定义博客文件夹名字”
cd "自定义博客文件夹名字"
npm install

GitHub服务器端

  • 在Github上创建一个仓库（repository），名字格式为，“ ‘你的GitHub名字’.github.io ”，这个仓库就会自动变成你的博客仓库
  • 配置仓库的SSH key到主机端，类比于在你的服务器与主机之间建立一个安全通道。这里不再赘述，可以参考《Github入门与实践》
  • 在主机上，打开自定义博客文件夹，这个根目录下有个“_config.yml”的配置文件，在“deploy”处添加自己的Github仓库的URL。至此，主机与服务器连接完成

博客构建

  • 博客的框架，装潢

    • 主题源

    • cd "自定义博客文件夹名字"
git clone "主题URL" themes/"这个主题名字"

  • 在根目录下的”_config.yml”配置文件中修改 “theme：’这个主题名字’ “

  • 参考已选择主题的documentation去美化装潢

  • 博客写作

    • 使用Markdown进行写作，Markdown的用法我会在下次更新时补上作为我个人练习

    • hexo new "新文章标题"          #生成新的博文
hexo clean                   #清空之前的配置，不然可能会更新不了页面
hexo g                       #生成新的配置
hexo s                       #本地调试
hexo d                       #部署到服务器端

    • 关于照片保存问题，我的照片是保存在我个人的亚马逊云S3上，而且没有经过水印处理，这些问题我会慢慢摸索

后记

我简单地描述了用Github Page和Hexo搭建博客的方法，因每个人电脑配置不同，可能还需要其他的一些操作，可以去百度或者谷歌搜一搜。我的方法仅供参考，如果有技术错误或者表述错误，实在抱歉，可以私信我进行技术交流，非常感谢。

最后附一张多大冬天的照片，测试一下照片能不能导入博客。:)
